title: StudioVerb
description: Original digital room/plate reverb model for Greybound.
---

StudioVerb is Greybound's original digital reverb pedal. It is not a clone and
not a graybox circuit model. The target is a practical studio-style reverb with
multiple algorithms that can sit after delay/spring in maintained rigs.

## Intended Role

- `room`: shorter, denser ambience for general use.
- `plate`: longer and smoother synthetic tail with less early-room emphasis.
- Default placement is `fx_loop` or `post_amp`.
- It belongs in `all-nox` and may be used in `grey-nox` as a creative room
  layer, but do not treat it as a real graybox reference.

## Controls

- `algorithm`: `room` or `plate`.
- `decay`: feedback intensity and tail persistence.
- `size`: delay-length scale.
- `pre_delay_ms`: dry-to-wet onset delay, clamped to 0-120 ms.
- `diffusion`: early reflection and FDN input density.
- `tone`: wet-path brightness.
- `low_cut`: removes low-frequency buildup before the tank.
- `mod_depth`: slow delay modulation depth.
- `mix`: wet/dry blend.

## Current DSP Approximation

The implementation uses:

- source/load input boundary and input coupling,
- low-cut high-pass before the reverb network,
- predelay,
- four short early reflection delays,
- 8-line feedback delay network mixed through a Hadamard matrix,
- per-line damping,
- slow line modulation,
- tone-dependent wet filtering,
- conservative wet/dry output scaling,
- 1 MOhm input impedance and 1 kOhm output impedance.

The model exposes meaningful voltage/input/output boundaries but does not map
to a physical schematic. Validate it as a designed digital effect.

- CLI/TUI display and runtime numeric control updates,

Evidence:

- `cargo test -p greybound studioverb --lib`: passed.
- `cargo test -p greybound rig::tests --lib`: passed.
- `cargo test -p greybound-cli`: passed.
- `cargo build --release`: passed.
- Isolated StudioVerb room render on `reverb-diagnostics.wav`: evaluation
  verdict `pass`, no near-clips, no hard clips, wet component `-89.40 dBFS`
  RMS, wet-to-dry `-23.78 dB`, wet component peak `-36.98 dBFS`, DC offset
  `-118.06 dBFS`, early centroid `10003.7 Hz`, mid centroid `4018.3 Hz`, late

- `all-nox` monitor render produced zero xruns, but the full-board output still
  clips heavily because every pedal is active. Treat that as the known all-board
  gain-staging problem, not as isolated StudioVerb instability.
- `grey-nox` now includes StudioVerb after Springfield with a conservative
  `mix = 0.10`. A 10-second monitor render to
  `target/greybound-grey-nox-studioverb-monitor.wav` produced zero xruns, zero
  near-clips, and zero hard clips; observed output RMS stayed around
  `-15` to `-18 dBFS`, with peaks below `-4.3 dBFS`.

## 2026-06-11 Audibility Retune

User feedback on the first `grey-nox` integration was that StudioVerb produced
no audible difference. The first evidence supported that complaint: the
isolated diagnostics documented a very conservative wet component and
`grey-nox` used only `mix = 0.10`, after an already active Springfield spring
layer.

Decision:

- raise StudioVerb room/plate input, early, and output wet gains while keeping
  feedback limits unchanged,
- add an algorithm-specific final wet gain (`room = 1.65`, `plate = 1.45`),
- move the default mix from `0.18` to `0.24`,
- move maintained rig mixes to `grey-nox = 0.22` and `all-nox = 0.24`.

Validation:

```bash
cargo test -p greybound studioverb --lib
cargo test -p greybound rig::tests --lib
cargo build --release
target/release/greybound-cli --rig rigs/grey-nox.json5 --input-wav "lab/references/tone3000-inputs/Brit - Guitar.wav" --output-wav target/greybound-grey-nox-studioverb-audible-monitor.wav --render-seconds 10 --sample-rate 48000 --period-size 16 --ir lab/references/tone3000-irs/celestion.wav --monitor
target/release/greybound-cli --rig target/grey-nox-studioverb-bypassed.json5 --input-wav "lab/references/tone3000-inputs/Brit - Guitar.wav" --output-wav target/greybound-grey-nox-studioverb-bypassed-monitor.wav --render-seconds 10 --sample-rate 48000 --period-size 16 --ir lab/references/tone3000-irs/celestion.wav --monitor --monitor-log target/greybound-grey-nox-studioverb-bypassed-monitor.log
uv --project lab run greybound-lab compare-wav --candidate target/greybound-grey-nox-studioverb-audible-monitor.wav --reference target/greybound-grey-nox-studioverb-bypassed-monitor.wav --report target/greybound-grey-nox-studioverb-vs-bypassed.md
```

Evidence:

- `cargo test -p greybound studioverb --lib`: passed.
- `cargo test -p greybound rig::tests --lib`: passed.
- `cargo build --release`: passed.
- Active `grey-nox` monitor render produced zero xruns, zero near-clips, and
  zero hard-clips; observed peak stayed below about `-4.5 dBFS`.
- Active-vs-bypassed comparison produced null residual `-44.22 dBFS`, residual
  relative to reference `-26.48 dB`, weighted guitar-band log-spectral distance
  `4.07 dB`, and only `0.225 dB` candidate gain correction. This confirms an
  audible time-domain change without relying on a simple loudness bump.

Remaining assumptions:

- The retune is still a designed digital effect, not a clone target.
- The active-vs-bypassed comparison uses guitar DI integration material, so it
  proves audibility in `grey-nox` but not detailed room realism.
